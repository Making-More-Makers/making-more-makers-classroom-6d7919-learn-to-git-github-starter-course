## ğŸ“– Course Information

**Course Name**: Making More Makers  
**Duration**: 6 Days (Intensive)  
**Format**: Hands-on Workshop  
**Prerequisites**: Basic programming knowledge (helpful but not required)

## ğŸ¯ Learning Objectives

By the end of this course, students will be able to:

1. **Understand Maker Culture**
   - Explain the principles and values of the maker movement
   - Identify opportunities for maker-based problem solving

2. **Hardware Development**
   - Program Arduino and ESP32 microcontrollers
   - Interface with various sensors and actuators
   - Integrate AI capabilities with ESP32

3. **Software Tools**
   - Use Git and GitHub for version control
   - Create documentation using Markdown
   - Deploy project websites using GitHub Pages

4. **Design Skills**
   - Design and prototype 3D models using TinkerCAD
   - Create functional enclosures for electronics projects

5. **Project Management**
   - Plan and execute a complete maker project
   - Work effectively in teams
   - Present technical work to diverse audiences

### Day 1: Introduction to Maker Culture & Arduino Fundamentals

**Morning Session (9:00 AM - 12:00 PM)**
- 9:00 - 10:00: Welcome & Maker Culture Introduction
  - What is a maker? History and philosophy
  - Examples of impactful maker projects
  - The maker mindset: iterate, share, collaborate
- 10:00 - 10:30: Course Overview
  - Course structure and expectations
  - Deliverables and timeline
  - Introduction to GitHub Classroom
- 10:30 - 10:45: Break

  - Grading rubric explanation

## ğŸ¤ Course Policies

### Attendance
- Full attendance expected for all sessions
- Notify instructor in advance if absence unavoidable
- Missed sessions must be made up through recorded materials

### Collaboration
- Team projects: all members must contribute
- Individual assignments: your own work, but discussion encouraged
- Proper attribution required for external code/resources

### Late Submissions
- Assignments due at specified times
- Late penalty: 10% per day
- Maximum 3 days late accepted
- Extensions granted for documented emergencies

### Academic Integrity
- Original work required
- Cite all sources and inspirations
- Open source code usage permitted with attribution
- Plagiarism results in zero grade

### Lab Safety
- Follow all safety guidelines
- Report damaged equipment immediately
- Clean up workspace after each session
