### 9:30 - 12:00: Project Presentations

**Format**: 10-minute presentation + 5-minute Q&A per team

**Presentation Order**: (Determined by instructors)

**What to Present**:

**1. Introduction** (~1 minute)
- Team name and members
- Project title
- Attention-grabbing opening

**2. Problem Statement** (~2 minutes)
- What problem are you solving?
- Why is it important?
- Who are the users?
- Real-world context

**3. Solution Overview** (~2 minutes)
- Your approach
- How it works (high level)
- Key features
- What makes it unique?

**4. Technical Implementation** (~2 minutes)
- Hardware components used
- Software architecture
- Interesting technical challenges
- How you solved them
- Technical decisions made

**5. Live Demonstration** (~2 minutes)
- Show the project working!
- Highlight key features
- User interaction
- (Backup video if live demo fails)

**6. Reflection & Future Work** (~1 minute)
- What you learned
- Challenges overcome
- What you'd improve
- Potential next steps

**Q&A Session** (5 minutes)
- Audience questions
- Instructor questions
- Technical clarifications
- Design decisions explanation

---
