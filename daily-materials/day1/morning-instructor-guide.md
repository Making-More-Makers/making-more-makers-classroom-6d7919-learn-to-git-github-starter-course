# Day 1 Morning: Instructor Execution Guide
# 第1天上午：教师执行指南

**总时长**: 90-120分钟  
**核心目标**: 建立团队、理解Maker文化、启动GitHub记录

---

## 📋 课前准备清单（提前30分钟）

### 物理环境
- [ ] 投影仪和屏幕测试完成
- [ ] 白板准备（擦干净 + 马克笔3支）
- [ ] 便利贴准备（每色50张 × 3色）
- [ ] 桌椅按分组排列（3-5人/组，共4-6组）
- [ ] Spaghetti Challenge材料按组摆放
- [ ] 计时器准备（可见的大屏倒计时）
- [ ] 测量工具准备（米尺或卷尺）

### 数字环境
- [ ] PPT打开并测试
- [ ] GitHub Classroom链接已生成
- [ ] 课程Wiki/资料链接已测试
- [ ] 备用材料准备（PDF版讲义）

### 个人准备
- [ ] 讲师台词熟悉一遍
- [ ] 水杯、马克笔在手边
- [ ] 手机设为静音（但保留计时功能）
- [ ] 深呼吸，微笑准备

### 环节1：开场欢迎（5-7分钟）
**时间**: 9:00 - 9:07  
**目标**: 建立期待、明确承诺

#### 🎤 讲师台词（可直接使用）

**开场（1分钟）**

> "大家好，欢迎来到 Making More Makers 创客成长营。
> 
> 在接下来的六天里，你会做出你人生中的第一个完整创客项目。
> 
> 它可能是一个 AI 语音助手、一个情绪灯、一个智能盒子，
> 也可能是你今天早上还没想到的酷东西。"

**[停顿2秒，环顾全场]**

**课程承诺（2分钟）**

> "在这个营里，我向你保证三件事：
> 
> **第一**，你会动手做出真实的东西。不是PPT，不是作业，是真实的、能用的、你可以拿回家的作品。
> 
> **第二**，你会失败很多次。但每次失败都会让你离成功更近一步。这里最鼓励的就是试错。
> 
> **第三**，你会认识一群和你一样热爱创造的人。这些人，可能会成为你未来的合作伙伴。"

**今天上午的三件事（2分钟）**

> "今天上午我们要做三件事：
> 
> **① 认识彼此** — 用一个特别的方式
> 
> **② 通过一个混乱但好玩的工程挑战** — 建立团队氛围
> 
> **③ 开启我们的 Maker 精神** — 理解为什么我们在这里
> 
> 之后的所有项目我们都会用 GitHub Classroom 来记录与追踪。
> 
> 所以你的创客旅程，现在就开始了。"

**[投影切换到"今天上午议程"页面]**

Making More Makers

Day 1 Morning

- 如果学生迟到：先开始，迟到学生加入时简短欢迎，稍后私下补充信息
- 如果投影失败：口述议程，继续进行
- 如果时间紧张：可压缩到5分钟（只讲核心承诺）

### 环节2：学生自我介绍（20-30分钟）
**时间**: 9:07 - 9:37  
**目标**: 破冰、了解学生、收集项目想法

#### 🎤 引导话术

**介绍规则（2分钟）**

> "现在我们要快速认识彼此。
> 
> 但不是传统的自我介绍——不需要说你的专业、你的GPA。
> 
> 我们用**'三件事自我介绍法'**。
> 
> 每个人用3句话介绍自己，每句话回答一个问题。"

**[投影切换到"三件事"页面]**

#### 📊 PPT内容
```
Slide 3: 三件事自我介绍
━━━━━━━━━━━━━━━━━━━
每人用 3 句话介绍自己：

① 你是谁？（名字 + 爱好）

② 为什么来参加这个创客营？

③ 你想做的项目是什么？（一句话）

⏱ 每人 20-30 秒
━━━━━━━━━━━━━━━━━━━
```

#### 📝 操作步骤

**Step 1: 示范（1分钟）**

> "我先来示范：
> 
> ① 我是Peter，我喜欢折腾各种电子玩具。
> 
> ② 我来做讲师是因为我想看到更多人把想法变成现实。
> 
> ③ 我最近在做一个用 ESP32 + AI 的智能对话盒子。
> 
> 就这么简单。现在从第一位同学开始。"

**Step 2: 学生轮流介绍（18-25分钟）**

- **你的任务**：
  1. 每个学生说完后，给予正向评价
  2. 将学生提到的**项目关键词**写在便利贴上
  3. 把便利贴贴到白板上（按类别排列）

- **正向评价示例**：
  - "这个项目非常适合用 ESP32！"
  - "这个想法很有创造力，后面我们会教你怎么做。"
  - "很好，这个可以加入 AI 模块。"
  - "有意思！这个方向很实用。"

**Step 3: 总结（2分钟）**

> "太好了！我看到了很多有趣的想法。
> 
> [指着白板上的便利贴]
> 
> 我们有智能硬件、互动艺术、实用工具、游戏装置……
> 
> 这些想法，在下午的时候会进入 GitHub Classroom。
> 
> 每个人会建立自己的 Maker Profile，记录你的创客身份。"

#### 📋 便利贴分类建议
```
白板区域划分：
┌─────────────────────┐
│  智能硬件  │  互动艺术  │
├─────────────────────┤
│  实用工具  │  游戏娱乐  │
├─────────────────────┤
│  健康监测  │  其他想法  │
└─────────────────────┘
```

#### ⚠️ 应急预案
- 如果有学生不知道做什么：鼓励说"我还在探索，但我对XX感兴趣"
- 如果有学生说得太长：友好打断"很好！先记住这个，后面我们深入聊"
- 如果时间紧张：可限制为每人15秒

### 环节3：Spaghetti Marshmallow Challenge（25-35分钟）
**时间**: 9:37 - 10:12  
**目标**: 团队协作、工程思维启蒙、创造快乐记忆

#### 🎤 活动介绍（2分钟）

> "接下来我们要做的是一个经典的 Maker 工程挑战。
> 
> 它源自设计公司 IDEO，全世界的创客营都在用它。
> 
> 挑战名字叫：**Spaghetti Marshmallow Challenge**
> 
> 目标很简单：用最简单的材料，搭出一个最高、能够站起来的结构。
> 
> 每组只有 **12 分钟**。"

**[投影切换到Challenge规则页]**

#### 📊 PPT内容
```
Slide 4: Spaghetti Marshmallow Challenge
━━━━━━━━━━━━━━━━━━━━━━━━━━━
挑战规则：

每组材料：
• 20 根 Spaghetti（意大利面）
• 1 个 Marshmallow（棉花糖）
• 1m 细绳
• 1 把剪刀

时间：12 分钟

目标：
搭建一个能自立的结构
棉花糖必须在最顶端
手离开后能独立站立 5 秒

最高者获胜 🏆
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 📝 详细操作流程

**Step 1: 分组（2分钟）**

> "现在我们分组。每组 3-5 人。
> 
> [指定分组方式]：
> - 按座位区域分组，或
> - 倒数 1-2-3-4，相同数字的组队
> 
> 每组给自己起一个队名，大声喊出来！"

**Step 2: 发放材料（1分钟）**

- 助教或讲师给每组发放材料包
- 确认每组材料完整
- 强调：**现在不能开始！**

**Step 3: 规则确认（1分钟）**

> "再强调几点：
> 
> ① 可以折断 Spaghetti，可以用绳子绑
> 
> ② 棉花糖**不能分开**，必须整个在顶端
> 
> ③ 最后测量时，手离开，结构能独立站 5 秒
> 
> ④ 12 分钟倒计时，到时间立刻停手
> 
> 准备好了吗？"

**Step 4: 开始挑战（12分钟）**

> "3... 2... 1... 开始！"

**[启动大屏计时器]**

**你在这12分钟做什么**：
1. **观察团队动态**：
   - 谁在领导？
   - 谁在动手？
   - 谁在思考策略？
   - 有冲突吗？

2. **适度鼓励**：
   - "加油！时间过半了！"
   - "记得测试一下结构稳定性！"
   - "还有3分钟！"

3. **拍照记录**：
   - 拍下团队协作的精彩瞬间
   - 后面可以用于总结

**Step 5: 停止与测量（3-5分钟）**

> "时间到！所有人停手！"

- 用米尺逐组测量高度（从桌面到棉花糖最高点）
- 记录在白板上
- 确认结构能自立5秒

**Step 6: 颁奖（1分钟）**

> "恭喜 [队名] 获得最高结构奖！
> 
> 你们搭了 [XX] 厘米！
> 
> 所有组都做得很棒，给自己鼓掌！"

**[拍照留念]**

#### 🧠 核心总结（5分钟）— **最重要的部分**

> "好，现在大家坐下来，我们来聊聊刚才发生了什么。
> 
> 这不只是一个游戏。
> 
> 这个活动包含了**所有工程思维的核心要素**：

**[投影切换到总结页]**

```
Slide 5: Spaghetti Challenge = 工程思维
━━━━━━━━━━━━━━━━━━━━━━━━━━━
刚才你们经历了什么？

✓ 设计（需要想策略）
✓ 材料选择（用什么、怎么用）
✓ 试错（很多尝试失败了）
✓ 迭代（一次次调整）
✓ 时间压力（12分钟很紧张）
✓ 团队合作（谁都不能单干）

这就是 **Maker 精神**：
做中学、快速试错、不断迭代、小步快跑
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

> "在接下来的项目里，你会一次次重复这个过程。
> 
> 设计 → 做 → 测试 → 失败 → 改进 → 再试
> 
> Maker 不怕失败，因为每次失败都是数据。
> 
> 好，我们休息10分钟。回来后，我们聊 Maker 文化。"

#### ⚠️ 应急预案
- 如果材料不够：提前准备备用包
- 如果有组完全失败：表扬尝试，强调"失败是学习的一部分"
- 如果时间不够：压缩到20分钟（游戏10分钟，总结3分钟）

#### 🚨 常见问题现场处理（非常实用！）

**问题1：学生打不开 QR 码**
- **解决**: 提供短链接（大声念出来）
- **备用**: 在聊天室/黑板写下完整链接

**问题2：学生没有 GitHub 账号**
- **解决**: 引导快速注册（邮箱 + 密码即可）
- **提示**: "用你常用的邮箱，30秒就能注册好"

**问题3：学生加入了错误账户**
- **解决**: 教他们登出（Sign Out）再用正确账号登录
- **位置**: GitHub 右上角头像 → Sign Out

**问题4：学生完全卡住**
- **解决**: 走到他们电脑旁边，帮他们点击
- **原则**: 不要让一个学生拖慢全班进度

**问题5：有学生已经完成，在等待**
- **解决**: "已经加入的同学可以先浏览 Assignment 的模板内容"
- **或**: "可以在脑子里想想你的 Maker Profile 要怎么写"

---

#### 💡 关键原则

**不要让学生互相等待**
- 让速度快的先进入 Assignment 模板浏览
- 你继续引导慢的学生
- 这是异步进度管理

**主动走动帮助**
- 看到学生卡住，立即走过去
- 手把手点击比远程指挥更有效
- 这10分钟你需要"巡场"

**保持耐心和鼓励**
- "没关系，第一次都会这样"
- "很好！你已经成功了一半"
- "太棒了，你是第X个加入的！"
